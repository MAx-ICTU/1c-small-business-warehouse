# Демо-сценарий для собеседования (10-15 минут)

## 1. Старт и структура (1-2 минуты)
- Показать подсистемы: `НСИ`, `Закупки`, `Продажи`, `Склад`, `Отчеты`, `Администрирование`, `Интеграция`.
- Кратко проговорить архитектуру:
  - Документы движения + регистры накопления.
  - Периодический регистр цен.
  - Общие модули для логики запасов и цен.
  - СКД-отчеты.
  - Роли.

## 2. НСИ и цены (2 минуты)
- Открыть `Номенклатура`, `Склады`, `Контрагенты`, `ВидыЦен`.
- Показать, что в регистре `ЦеныНоменклатуры` задаются цены.

## 3. Сквозной бизнес-кейс (4-5 минут)
1. Провести `ПоступлениеТоваров` на склад.
2. Создать и провести `ЗаказКлиента` с резервом.
3. Создать и провести `РеализацияТоваров` по заказу-основанию.
4. Показать `ПеремещениеТоваров` и `СписаниеТоваров`.

Что озвучить:
- Контроль остатков при расходных документах.
- Контроль доступности резерва в заказе.
- Снятие резерва при реализации.
- Понятные сообщения пользователю при дефиците.

## 4. Отчеты (2-3 минуты)
- `ОстаткиТоваровПоСкладам`
- `ПродажиЗаПериод`
- `ТоварыВРезерве`
- `ТопТоваров`

Что озвучить:
- Отчеты построены на СКД.
- Параметры данных отделены от пользовательского отбора.

## 5. Интеграция JSON (1-2 минуты)
- Открыть обработку `ЗагрузкаЗаказовИзJSON`.
- Загрузить файл `samples/orders/order-001.json`.
- Показать созданный документ `ЗаказКлиента`.

## 6. Печатные формы (1 минута)
- Вызвать формирование печатной формы для:
  - `ЗаказКлиента`
  - `РеализацияТоваров`

## 7. Права (1 минута)
- Показать роли:
  - `Менеджер`
  - `Кладовщик`
  - `Администратор`
- Кратко пояснить разграничение по задачам.

## 8. Сильные стороны проекта для работодателя
- Прикладная архитектура, а не “учебный конструктор”.
- Реальная логика проведения документов с валидациями.
- Работа с регистрами накопления и периодическим регистром сведений.
- Вынос бизнес-логики в общие модули.
- Отчеты на СКД и базовая интеграция JSON.
- Автоматизированный процесс выгрузки/загрузки конфигурации и GitHub sync.
