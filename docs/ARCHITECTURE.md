# Архитектура и объектная модель

## Подсистемы
- `НСИ`
- `Закупки`
- `Продажи`
- `Склад`
- `Отчеты`
- `Администрирование`
- `Интеграция`

## Основные объекты

### Справочники
- `Номенклатура`
- `Склады`
- `Контрагенты`
- `Сотрудники`
- `ВидыЦен`

### Документы
- `ПоступлениеТоваров`
- `ЗаказКлиента`
- `РеализацияТоваров`
- `ПеремещениеТоваров`
- `СписаниеТоваров`

### Регистры
- Накопления `ОстаткиТоваров` (вид: Остатки).
- Накопления `РезервыТоваров` (вид: Остатки).
- Сведений `ЦеныНоменклатуры` (периодический).

## Логика движений
- `ПоступлениеТоваров`: приход в `ОстаткиТоваров`.
- `ЗаказКлиента`: приход в `РезервыТоваров`.
- `РеализацияТоваров`: расход из `ОстаткиТоваров` и расход из `РезервыТоваров`.
- `ПеремещениеТоваров`: расход со склада-источника и приход на склад-получатель.
- `СписаниеТоваров`: расход из `ОстаткиТоваров`.

## Технические блоки
- Общие модули для управления запасами, ценами и сервисными функциями.
- Отчеты на СКД.
- Обработка загрузки данных из JSON.
- Печатные формы для ключевых документов.
